# Workspace

## Overview

pnpm workspace monorepo using TypeScript. This is the **Trada** platform — an AI-driven cryptocurrency trading app where users rent AI trading agents, fund a USDT wallet, and let agents execute trades on their behalf.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)
- **Frontend**: React 19 + Vite + TanStack Query + shadcn/ui + Tailwind CSS v4
- **Routing (client)**: wouter
- **Auth**: JWT (`trada_token` in localStorage) + bcryptjs

## Key Commands

- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- `pnpm --filter @workspace/api-server run dev` — run API server locally

## Architecture

```
lib/
  api-spec/         # OpenAPI spec (openapi.yaml) — source of truth for all endpoints
  api-zod/          # Zod schemas generated from OpenAPI (only exports from ./generated/api)
  api-client-react/ # TanStack Query hooks generated by Orval
  db/               # Drizzle ORM schema + migrations
    src/schema/     # users, wallets, depositAddresses, transactions, withdrawalRequests,
                    # agents, rentals, trades, notificationPreferences, systemAlerts, adminAuditLog

artifacts/
  trada/            # React frontend
    src/pages/      # dashboard, wallet, agents, rentals, trades, settings, admin-*, login, register
    src/lib/auth.tsx # AuthContext, AuthProvider, useAuth, ProtectedRoute
    src/components/ # layout, ui/* (shadcn components)
  api-server/
    src/routes/     # auth, wallet, agents, rentals, trades, notifications, dashboard, admin
    src/middlewares/# auth.ts (requireAuth, requireAdmin)
    src/app.ts      # Express app setup
    src/index.ts    # Server entry
```

## Auth Flow

- JWT stored in `localStorage` as `trada_token`
- Custom axios instance (`lib/api-client-react/src/client.ts`) reads it via `setAuthTokenGetter`
- `requireAuth` middleware validates JWT on every protected route
- `requireAdmin` also checks `role === 'admin'`

## API Routes

All routes are under `/api`:

| Group | Endpoints |
|-------|-----------|
| Auth | `POST /auth/register`, `POST /auth/login`, `GET /auth/me` |
| Wallet | `GET /wallet`, `GET /wallet/transactions`, `GET /wallet/deposit-address`, `POST /wallet/withdraw` |
| Agents | `GET /agents`, `GET /agents/:id` |
| Rentals | `GET /rentals`, `GET /rentals/:id`, `POST /rentals`, `POST /rentals/:id/cancel` |
| Trades | `GET /trades`, `GET /trades/:id` |
| Notifications | `GET /notifications/preferences`, `PUT /notifications/preferences` |
| Dashboard | `GET /dashboard/summary`, `GET /dashboard/activity`, `GET /dashboard/performance` |
| Admin | `GET /admin/dashboard`, `GET /admin/users`, `GET /admin/withdrawals`, `POST /admin/withdrawals/:id/approve`, `POST /admin/withdrawals/:id/reject`, `GET /admin/alerts`, `POST /admin/alerts/:id/resolve`, `POST /admin/rentals/:id/pause`, `POST /admin/rentals/:id/resume`, `POST /admin/rentals/:id/reset-circuit-breaker` |

## Notes

- `lib/api-zod/src/index.ts` must only `export * from "./generated/api"` — do NOT add type re-exports (causes name conflicts with Zod schemas)
- DB schema uses varchar UUIDs for all primary keys (auto-generated via `gen_random_uuid()`)
- Password hashing uses bcryptjs with 12 rounds
- Frontend pages use generated hooks from `@workspace/api-client-react`

See the `pnpm-workspace` skill for workspace structure, TypeScript setup, and package details.
